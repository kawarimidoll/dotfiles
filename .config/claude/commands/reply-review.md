# /reply-review - レビューコメント返信コマンド

`/address` でレビュー対応を行った後、各レビューコメントに対して対応した旨（または対応不要と判断した旨）を返信します。

## 使用方法

```
/reply-review [PR番号 | URL]
```

### 引数の解釈

- **無引数**: 直前の会話に含まれるレビュー対応サマリーとPR情報を使用する
- **数値のみ**: PR番号として解釈
- **github.com を含むURL**: GitHub PR URLとして解釈し、PR番号を抽出

## 前提条件

- `/address` による対応が完了していること
- 直前の会話にレビュー対応サマリー（対応した指摘 / 対応しなかった指摘）が含まれていること

## 動作フロー

1. PR番号を特定する（引数、または直前の会話から推測）
2. 直前の会話からレビュー対応サマリーを取得する
3. PRのレビューコメント一覧を取得する
   - `gh api repos/{owner}/{repo}/pulls/{番号}/comments` — インラインコメント
   - `gh api repos/{owner}/{repo}/pulls/{番号}/reviews` — レビュー本文
4. 各コメントに対して、対応サマリーを元に返信を作成する
5. `gh api` を使って各コメントに返信を投稿する

## 返信の形式

### 対応した指摘への返信

```
修正しました。{修正内容の簡潔な説明}

> 🤖 Claude Code
```

### 対応不要と判断した指摘への返信

```
確認しましたが、以下の理由により現状維持としました。

{対応不要と判断した理由}

> 🤖 Claude Code
```

## 返信APIの使い方

### インラインコメントへの返信

```bash
gh api repos/{owner}/{repo}/pulls/{番号}/comments/{comment_id}/replies \
  -f body="返信内容"
```

### レビュー本文へのコメント（レビュー全体に返信がある場合）

```bash
gh api repos/{owner}/{repo}/issues/{番号}/comments \
  -f body="返信内容"
```

## 注意事項

- 返信前に必ずユーザーに返信内容の一覧を提示し、承認を得てから投稿する
- 既に返信済みのコメントには重複して返信しない
